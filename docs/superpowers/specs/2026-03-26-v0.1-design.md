## 1. 版本目标

用户能在浏览器里跑通一个 Feature 从 brainstorming 到 merge 的完整开发流程。

交付标准：
- 用户在 Web UI 创建 Project（配置 Git 仓库 + SSH 凭据）
- 用户手动创建 Feature，填写标题和描述
- 点击"启动"后，系统自动驱动 Feature 经历完整状态机
- 4 个人工介入节点均可在 Web UI 操作（确认 / 拒绝 / 回溯）
- 实时进度通过 WebSocket 推送到浏览器
- 飞书单向通知：状态变更时推送消息（不需要在飞书操作）
- 最终 PR 自动 merge，worktree 清理

## 2. 功能范围

### 包含

| 功能 | 说明 |
|------|------|
| Project 管理 | 创建 / 编辑 Project，配置 ssh_url、默认 Agent、SSH 私钥 |
| Feature 手动创建 | 填写标题 + 描述，手动启动 |
| Feature 状态机 | brainstorming → planning → implementing → testing → reviewing → approved → verifying → merged |
| 人工介入节点 | 共 4 次（确认 brainstorming、确认 plan、确认测试结果、Approve/Reject review） |
| 可回溯 | testing 失败 / review reject / verification 失败时可回到 brainstorming 重试（增量） |
| failed / archived 终态 | retry_count 超限进入 failed，人工可重置或放弃 |
| Claude Code Agent | 唯一 Agent 实现，执行 brainstorm / plan / implement |
| Git Manager | clone、branch、commit、PR 创建、rebase、merge、worktree 清理 |
| Web UI | Next.js，Project 管理 + Feature 管理 + 实时进度 |
| 飞书单向通知 | 状态变更时推送 Webhook 消息，不需要双向交互 |

### 不包含

| 功能 | 推迟版本 |
|------|----------|
| FeaturePlanner / 需求管理 | v0.2 |
| 多 Feature 并行执行 | v0.3 |
| 飞书双向交互（消息卡片操作） | v0.4 |
| 多 Agent 支持（Codex 等） | v0.5 |

## 4. 模块设计

v0.1 涉及以下模块：

### 4.1 Feature Executor

驱动单个 Feature 的完整状态机，是系统核心。

职责：
- 接收"启动 Feature"指令，创建 Celery 任务
- 按状态机顺序调用各阶段执行器
- 每个阶段完成后更新 Feature 状态，写入 FeatureExecution 记录
- 遇到人工介入节点时暂停，等待 Approval Gateway 的信号
- 处理回溯逻辑：将上次结果 + 失败原因传入下次 brainstorming

状态机流转：

```
pending
  ↓ [人工] 启动
brainstorming  ← 回溯入口（携带上次结果 + 失败原因）
  ↓ [人工介入 1] 确认 brainstorming 结果
planning
  ↓ [人工介入 2] 确认实现计划
implementing
  ↓
testing
  ↓ [人工介入 3] 确认测试结果
    ↓ 通过                ↓ 失败：回溯 or 忽略继续
reviewing
  ↓ [人工介入 4] Approve / Reject
    ↓ Approve             ↓ Reject：回溯
approved
  ↓ 自动触发
verifying
  ↓ 通过                  ↓ 失败：回退到 approved，人工选择重新触发 or 回溯
merged
```

回溯规则：
- `retry_count` 递增，超过 `max_retries`（默认 3）进入 `failed`
- brainstorming 阶段接收 `previous_result` + `failure_reason`，增量分析
- implementing 阶段在现有 worktree 代码上修改，不重置 branch

### 4.2 Agent Executor

管理 Claude Code Agent 的生命周期。

职责：
- 封装 Claude Code CLI 调用（子进程）
- 实时捕获 stdout/stderr，通过 WebSocket 推送日志流
- 将 Agent 输出解析为结构化结果（BrainstormResult / Plan / ImplementResult）
- 超时控制和错误处理

接口：

```python
class ClaudeCodeAgent:
    async def brainstorm(self, feature: Feature, previous: BrainstormResult | None = None, failure_reason: str | None = None) -> BrainstormResult: ...
    async def plan(self, feature: Feature, brainstorm: BrainstormResult, previous_plan: Plan | None = None) -> Plan: ...
    async def implement(self, feature: Feature, plan: Plan, worktree_path: str) -> ImplementResult: ...
```

### 4.3 Git Manager

封装所有 Git 操作。

职责：
- `clone(ssh_url, target_dir)` — 使用配置的 SSH 私钥 clone 仓库
- `create_worktree(repo_path, branch_name)` — 创建独立 worktree
- `commit(worktree_path, message)` — 提交 Agent 产出的代码
- `create_pr(branch, title, body)` — 调用 Git 平台 API 创建 PR
- `rebase(worktree_path, base_branch)` — rebase 到最新主分支，冲突时抛出异常
- `merge_pr(pr_url)` — 合并 PR
- `cleanup_worktree(worktree_path)` — 删除 worktree 目录

SSH 私钥通过环境变量注入，Git 操作时写入临时文件，用完即删。

### 4.4 Review Engine

对 implementing 阶段产出的代码做 AI code review。

职责：
- 获取 Feature branch 的 diff（相对主分支）
- 调用 LLM 生成结构化 review 报告（summary + issues 列表）
- 写入 ReviewReport 表

### 4.5 Approval Gateway

处理所有人工介入节点。

职责：
- 提供 REST API 接收人工决策（approve / reject / retry / ignore）
- 验证当前 Feature 状态是否允许该操作
- 更新 Feature 状态，唤醒等待中的 Celery 任务

4 个介入节点：

| 节点 | 触发状态 | 可选操作 |
|------|----------|----------|
| 确认 brainstorming | brainstorming（等待确认） | approve（进入 planning）/ reject（重新 brainstorm） |
| 确认 plan | planning（等待确认） | approve（进入 implementing）/ reject（回到 brainstorming） |
| 确认测试结果 | testing（等待确认） | approve（进入 reviewing）/ retry（回到 brainstorming）/ ignore（忽略失败，进入 reviewing） |
| Approve/Reject review | reviewing（等待确认） | approve（进入 approved）/ reject（回到 brainstorming） |

### 4.6 Notification Hub

统一推送通知。

v0.1 支持两个渠道：
- **WebSocket**：实时推送 Feature 状态变更 + Agent 日志流到浏览器
- **飞书 Webhook**：状态变更时 POST 到配置的 Incoming Webhook URL，单向推送

消息格式（飞书）：
```json
{
  "msg_type": "text",
  "content": {
    "text": "[solo100] Feature「{title}」状态变更：{old_status} → {new_status}\n{detail_url}"
  }
}
```

## 5. 数据模型

v0.1 涉及以下表：

-- v0.1 飞书仅单向推送，通过全局环境变量 FEISHU_WEBHOOK_URL 配置，不存数据库

```sql
-- 项目
CREATE TABLE project (
    id                TEXT PRIMARY KEY,  -- UUID
    name              TEXT NOT NULL,
    ssh_url           TEXT NOT NULL,
    default_branch    TEXT NOT NULL DEFAULT 'main',
    ssh_key_env       TEXT NOT NULL,     -- 环境变量名，如 SSH_KEY_PROJECT_1，不存明文
    default_agent_id  TEXT,              -- 引用 agent_config.id，可为 NULL
    created_at        DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Feature
CREATE TABLE feature (
    id          TEXT PRIMARY KEY,  -- UUID
    project_id  TEXT NOT NULL REFERENCES project(id),
    title       TEXT NOT NULL,
    description TEXT NOT NULL,
    status      TEXT NOT NULL DEFAULT 'pending',
    -- status 枚举：pending | brainstorming | planning | implementing |
    --              testing | reviewing | approved | verifying | merged | failed | archived
    branch      TEXT,              -- feat/<id-prefix>-<title-slug>，启动时生成
    pr_url      TEXT,
    worktree_path TEXT,
    retry_count INTEGER NOT NULL DEFAULT 0,
    max_retries INTEGER NOT NULL DEFAULT 3,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Feature 各阶段执行记录
CREATE TABLE feature_execution (
    id             TEXT PRIMARY KEY,  -- UUID
    feature_id     TEXT NOT NULL REFERENCES feature(id),
    attempt_number INTEGER NOT NULL,  -- 第几次尝试（回溯时递增）
    stage          TEXT NOT NULL,
    -- stage 枚举：brainstorming | planning | implementing | testing | reviewing | verifying
    status         TEXT NOT NULL,     -- running | completed | failed
    result_json    TEXT,              -- 阶段产出，JSON 格式
    started_at     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    finished_at    DATETIME
);

-- Code Review 报告
CREATE TABLE review_report (
    id              TEXT PRIMARY KEY,  -- UUID
    feature_id      TEXT NOT NULL REFERENCES feature(id),
    attempt_number  INTEGER NOT NULL,
    ai_summary      TEXT,              -- AI review 摘要
    ai_issues_json  TEXT,              -- AI 发现的问题列表，JSON 格式
    human_decision  TEXT,              -- approve | reject，人工决定后填入
    decided_by      TEXT,              -- 操作人标识（v0.1 固定为 "human"）
    decided_at      DATETIME,
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Agent 配置（v0.1 只有 Claude Code，但保留扩展性）
CREATE TABLE agent_config (
    id          TEXT PRIMARY KEY,  -- UUID
    name        TEXT NOT NULL,
    type        TEXT NOT NULL DEFAULT 'claude_code',
    api_key_env TEXT NOT NULL,     -- 环境变量名，不存明文
    is_default  INTEGER NOT NULL DEFAULT 0,
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## 6. API 设计

### 6.1 REST API

**Project**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/projects` | 获取 Project 列表 |
| POST | `/api/v1/projects` | 创建 Project |
| GET | `/api/v1/projects/{id}` | 获取 Project 详情 |
| PUT | `/api/v1/projects/{id}` | 更新 Project |
| DELETE | `/api/v1/projects/{id}` | 删除 Project |

**Feature**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/projects/{project_id}/features` | 获取 Feature 列表 |
| POST | `/api/v1/projects/{project_id}/features` | 创建 Feature |
| GET | `/api/v1/features/{id}` | 获取 Feature 详情（含最新执行记录） |
| POST | `/api/v1/features/{id}/start` | 启动 Feature（pending → brainstorming） |
| POST | `/api/v1/features/{id}/archive` | 放弃 Feature（→ archived） |
| POST | `/api/v1/features/{id}/reset` | 重置重试（failed → brainstorming，retry_count 清零） |

**人工介入（Approval Gateway）**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/features/{id}/approve` | 通用 approve，驱动当前等待节点通过 |
| POST | `/api/v1/features/{id}/reject` | 通用 reject，回到 brainstorming |
| POST | `/api/v1/features/{id}/ignore-test-failure` | 忽略测试失败，继续进入 reviewing |
| POST | `/api/v1/features/{id}/retry-verification` | 重新触发 verification（approved 状态） |

**执行记录 & Review**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/features/{id}/executions` | 获取所有执行记录 |
| GET | `/api/v1/features/{id}/review` | 获取最新 ReviewReport |

**Agent 配置**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/agents` | 获取 Agent 配置列表 |
| POST | `/api/v1/agents` | 创建 Agent 配置 |
| PUT | `/api/v1/agents/{id}` | 更新 Agent 配置 |

### 6.2 WebSocket 端点

```
WS /ws/features/{feature_id}
```

服务端推送的消息类型：

```typescript
// 状态变更
{ type: "status_change", feature_id: string, old_status: string, new_status: string, timestamp: string }

// Agent 日志流（实时）
{ type: "log", feature_id: string, stage: string, line: string, timestamp: string }

// 阶段完成
{ type: "stage_complete", feature_id: string, stage: string, result: object, timestamp: string }

// 等待人工介入
{ type: "awaiting_approval", feature_id: string, stage: string, message: string, timestamp: string }

// 错误
{ type: "error", feature_id: string, stage: string, message: string, timestamp: string }
```

## 7. 前端页面结构

### 7.1 页面列表

```
/                          → 重定向到 /projects
/projects                  → Project 列表页
/projects/new              → 创建 Project
/projects/[id]/settings    → Project 设置（编辑 ssh_url、SSH 凭据、飞书 Webhook）
/projects/[id]/features    → Feature 列表页（该 Project 下所有 Feature）
/features/new?project=[id] → 创建 Feature
/features/[id]             → Feature 详情页（主操作页面）
/settings/agents           → Agent 配置页
```

### 7.2 关键页面和组件

**Feature 列表页 `/projects/[id]/features`**

- `FeatureList` — 展示 Feature 卡片列表，按状态分组或排序
- `FeatureCard` — 显示 Feature 标题、当前状态、retry_count、最后更新时间
- `CreateFeatureButton` — 跳转到创建页

**Feature 详情页 `/features/[id]`**（核心页面）

布局：左侧状态机进度 + 右侧内容区

- `FeatureStatusStepper` — 可视化状态机进度条，高亮当前阶段
- `ApprovalPanel` — 当前等待人工介入时显示，包含操作按钮（Approve / Reject / Ignore / Retry）
- `ExecutionLogViewer` — 实时滚动日志，WebSocket 接收 log 消息
- `BrainstormResultCard` — 展示 brainstorming 阶段产出
- `PlanCard` — 展示 planning 阶段产出（任务列表、文件变更计划）
- `TestReportCard` — 展示 testing 阶段产出（测试结果、通过率）
- `ReviewReportCard` — 展示 AI code review 报告（summary + issues 列表）
- `FeatureMetaPanel` — 显示 branch、PR 链接、retry_count、worktree_path

**WebSocket 集成**

Feature 详情页挂载时建立 WebSocket 连接，卸载时断开。接收到消息后：
- `status_change` → 更新页面状态，刷新 ApprovalPanel
- `log` → 追加到 ExecutionLogViewer
- `stage_complete` → 刷新对应结果卡片
- `awaiting_approval` → 显示 ApprovalPanel，高亮操作按钮

 |                      |                    |← 202 Accepted -------|

 |                      |                    |                      |-- clone repo

 |                      |                    |                      |-- call ClaudeCodeAgent.brainstorm()
 |                      |← WS: log ---------|←----- push log ------|
 |                      |← WS: log ---------|←----- push log ------|

 |                      |← WS: awaiting_approval ←--- push --------|
 |                      |                    |                      |  (等待人工确认)
 |-- 查看结果 --------→ |                    |                      |
 |-- POST /approve ---→ |                    |                      |
 |                      |-- POST /approve -→ |                      |
 |                      |                    |-- signal task -----→ |
 |                      |                    |                      |-- 进入 planning...
